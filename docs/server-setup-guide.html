Server Setup & Training Guide

Complete reference for configuring a Hetzner server, managing data, and running full model training.

⚡ Quick Start — setup.sh

One command does everything. Each step is skipped automatically if already done — safe to run on a fresh server or an existing one after a git pull.

Step 1 — Mount & formatFormat skipped if files exist OR filesystem detected. Mount skipped if already mounted.
Step 2 — DirectoriesSkipped if folders exist (mkdir -p)
Step 3 — Git clone/pullClones fresh or pulls if repo exists
Step 4 — Python venvSkipped if /root/venv exists
Step 5 — paths.yamlSkipped if file already exists
Step 6 — HF loginSkipped if HF_TOKEN not set
# Step 0 — export tokens FIRST (before running setup)

# GitHub token: required for private repos
# Get from: https://github.com/settings/tokens  (repo scope)
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"

# Hugging Face token: required for uploading models/watchlists to HF Hub
# Get from: https://huggingface.co/settings/tokens  (write access)
export HF_TOKEN="hf_xxxxxxxxxxxxxxxxxxxx"

# Fresh server — bootstrap from GitHub directly
curl -o setup.sh https://raw.githubusercontent.com/prabhuvictor85/ml-stock-predictor/master/scripts/server_setup.sh
bash setup.sh

# Existing server after git pull — re-run safely, skips completed steps
bash /root/ml-stock-predictor/scripts/server_setup.sh
🔑

Public repo? Skip GITHUB_TOKEN — plain HTTPS clone works without it.
Private repo? Generate at github.com/settings/tokens with repo scope.
HF Hub? Generate at huggingface.co/settings/tokens with write access. Skip if you don't upload to HF.
Neither token is ever written to disk or printed in logs.

⚠️

paths.yaml is never overwritten if it already exists. To regenerate it: rm /root/ml-stock-predictor/paths.yaml && bash setup.sh

✅

When it prints "Server setup complete!" — activate the venv and you're ready: source /root/venv/bin/activate

1. Create Server (Hetzner)

Recommended spec for full training (NSE: n_jobs=4, SP500: n_jobs=2):

SettingValueNotes
Server typeCCX338 vCPU, 32 GB RAM — minimum for SP500 full run
ImageUbuntu 22.04 LTSStable, well-supported
LocationNBG1 / FSN1Closest to your data sources
Volume100 GB+Attach at creation (see Section 2)
SSH keyYour public keyAdd before creation
⚠️

Do not store stock data or artefacts on the root disk — use the attached volume at /mnt/data. Root disk is lost when server is deleted.

2. Attach & Mount Volume

Create & attach in Hetzner Console

Go to console.hetzner.cloud → Volumes → Create Volume. Set size ≥ 100 GB, attach to your server.

Find the device name

ls /dev/disk/by-id/ | grep HC_Volume

Note the full path, e.g. /dev/disk/by-id/scsi-0HC_Volume_12345678

Format (brand new volume only)

# Run BOTH checks before formatting — skip if either returns data

# Gate 1: files/folders at mount point? → skip format
ls -A /mnt/data 2>/dev/null
# Gate 2: filesystem already on device? → skip format
blkid -o value -s TYPE /dev/disk/by-id/scsi-0HC_Volume_XXXXXXXX

# Only format if BOTH returned empty (truly blank new volume)
mkfs.ext4 /dev/disk/by-id/scsi-0HC_Volume_XXXXXXXX
🛑

Never skip these checks. Formatting a volume with existing data destroys it permanently. If either command returns output, do NOT format.

Mount

mkdir -p /mnt/data
mount -o discard,defaults /dev/disk/by-id/scsi-0HC_Volume_XXXXXXXX /mnt/data

Make mount permanent (survives reboot)

# Add to /etc/fstab
echo "/dev/disk/by-id/scsi-0HC_Volume_XXXXXXXX /mnt/data ext4 discard,nofail,defaults 0 0" >> /etc/fstab

Resize filesystem after volume resize

# After resizing volume in Hetzner console (no reboot needed)
resize2fs /dev/disk/by-id/scsi-0HC_Volume_XXXXXXXX
df -h /mnt/data  # confirm new size

Create directory structure

mkdir -p /mnt/data/artefacts/{nse_local,us_local}
mkdir -p /mnt/data/Learning_charts/{stock_data/{nse_data,us_stocks,tradingview},stock_lists}

Detach volume before shutdown

# 1. Shutdown server gracefully
sudo poweroff

# 2. In Hetzner console: Volumes → ⋮ → Detach
# Volume + all data is preserved and billed at ~€0.05/GB/month
# Server stops billing once deleted

3. Git Setup

Configure identity

git config --global user.email "your@email.com"
git config --global user.name "Your Name"
git config --global pull.rebase false  # use merge on pull

Clone the repo

# Public repo — no token needed
git clone https://github.com/prabhuvictor85/ml-stock-predictor.git /root/ml-stock-predictor

# Private repo — export token first, then clone
export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
git clone https://${GITHUB_TOKEN}@github.com/prabhuvictor85/ml-stock-predictor.git /root/ml-stock-predictor

Pull latest changes (existing server)

cd /root/ml-stock-predictor
git stash            # park local changes (paths.yaml etc.)
git pull origin master
git stash pop        # restore paths.yaml
⚠️

If you see "divergent branches": git stash → git reset --hard origin/master → git stash pop

Resolve common conflicts

# paths.yaml conflict (machine-specific — always keep ours)
git checkout --ours paths.yaml
git add paths.yaml

# Artefact files staged for deletion — unstage them
git restore --staged artefacts/

4. Python & Virtual Environment

Install Python dependencies

apt update && apt install -y python3 python3-venv python3-pip

Create and activate venv

python3 -m venv /root/venv
source /root/venv/bin/activate

# Add to .bashrc so it auto-activates on SSH login
echo "source /root/venv/bin/activate" >> ~/.bashrc

Install requirements

pip install --upgrade pip
pip install -r /root/ml-stock-predictor/requirements.txt

Install HF CLI

pip install huggingface_hub
hf auth login   # paste your HF write token

artefacts_root: /mnt/data/artefacts

8. When to Run Full Training

Trigger Cadence Command Priority
Quarterly refresh
Mar / Jun / Sep / Dec		 Every 3 months run_nse_local.py (no --skip_train) Routine
Feature drift alert
drift_monitor flags degradation		 Immediate Full retrain on latest data Urgent
Major market event
Rate shock, crash, regime change		 Immediate Full retrain with updated --as_of Urgent
New data source added
New features or universe change		 Once after change Full retrain from scratch As needed
Weekly watchlist refresh
Same model, fresh market data		 Weekly / Daily --skip_train only Routine
✅

Key distinction: --skip_train rescores existing models on fresh data (fast, minutes). Full training rebuilds models from scratch (slow, 3–5 hours). Run --skip_train weekly; full retrain quarterly.

9. Pre-training Checklist

Complete these steps before starting a full training run:

Server & Environment

  • Server is CCX33 (8 vCPU / 32 GB RAM) or larger
  • Volume mounted at /mnt/data — confirm with df -h /mnt/data
  • At least 30 GB free on /mnt/data for SP500 (NSE needs ~5 GB)
  • venv activated: source /root/venv/bin/activate
  • Latest code pulled: git pull origin master
  • paths.yaml exists and points to correct data directories

Data

  • Stock CSV files present in /mnt/data/Learning_charts/stock_data/
  • Stock list CSV present (e.g. constituentsi.csv)
  • Benchmark data present (^NSEI-1d.csv or ^GSPC-1d.csv)
  • Data is fresh — latest bar within last 5 trading days

Cleanup (avoid stale checkpoint issues)

  • Delete old Optuna HPO databases if restarting from scratch
  • Delete stale panel checkpoints if numpy version changed
  • Run pip cache purge if disk space is low
# Cleanup stale checkpoints (run if restarting from scratch)
rm -f /mnt/data/artefacts/nse_local/momentum/checkpoints/optuna_hpo.db
rm -f /mnt/data/artefacts/nse_local/reversal/checkpoints/optuna_hpo.db
rm -f /mnt/data/artefacts/nse_local/checkpoints/*.pkl

rm -f /mnt/data/artefacts/us_local/momentum/checkpoints/optuna_hpo.db
rm -f /mnt/data/artefacts/us_local/reversal/checkpoints/optuna_hpo.db
rm -f /mnt/data/artefacts/us_local/checkpoints/*.pkl

# Check free disk space
df -h /mnt/data

10. Run Full Training

💡

Always run inside tmux so the session survives SSH disconnection. Use Ctrl+B d to detach and tmux attach -t training to reconnect.

Start tmux session

tmux new -s training

# Tmux quick reference:
# Ctrl+B %     → split pane vertically
# Ctrl+B "     → split pane horizontally
# Ctrl+B →/←   → move between panes
# Ctrl+B z     → zoom/unzoom pane (full screen)
# Ctrl+B d     → detach (leave running)
# tmux attach -t training  → re-attach

NSE Full Training

source /root/venv/bin/activate
cd /root/ml-stock-predictor

python3 run_nse_local.py \
  --train_start 2010-01-01 \
  --as_of 2023-12-08 \
  --n_jobs 4 \
  2>&1 | tee /mnt/data/artefacts/nse_local/run_nse_full.log
# --n_folds is auto-computed from date range (2023 - 2010 - 2 warmup + 1 = 12 folds)
# Do NOT pass --n_folds manually unless you need to override.

# Expected time breakdown:
#   Feature engineering + targets : ~10 min (or instant from checkpoint)
#   Fold cache (11 folds, ~1yr each): ~20 min
#   HPO (25 trials × 4 parallel)   : ~25 min per mode
#   Final model training            : ~5 min per mode
#   Total (2 modes)                 : ~1.5–2 hours

SP500 Full Training (in separate tmux pane)

# Ctrl+B % to split, Ctrl+B → to move to new pane
source /root/venv/bin/activate
cd /root/ml-stock-predictor

python3 run_sp500_local.py \
  --train_start 2010-01-01 \
  --as_of 2023-12-08 \
  --n_jobs 2 \
  2>&1 | tee /mnt/data/artefacts/us_local/run_sp500_full.log
# --n_folds is auto-computed from date range (2023 - 2010 - 2 warmup + 1 = 12 folds)
# Do NOT pass --n_folds manually unless you need to override.
# --n_jobs 2 for SP500 (large folds — each trial needs 4 LightGBM threads)

# Expected time breakdown:
#   Feature engineering + targets : ~15 min (or instant from checkpoint)
#   Fold cache (11 folds, large)   : ~55–60 min
#   HPO (25 trials × 2 parallel)   : ~45–60 min per mode
#   Final model training            : ~10 min per mode
#   Total (2 modes)                 : ~3–4 hours

Weekly Watchlist Refresh (skip_train)

# Fast — just rescores existing models on fresh data
python3 run_nse_local.py \
  --skip_train \
  --as_of $(date +%Y-%m-%d) \
  2>&1 | tee /mnt/data/artefacts/nse_local/run_nse_skip.log

python3 run_sp500_local.py \
  --skip_train \
  --as_of $(date +%Y-%m-%d) \
  2>&1 | tee /mnt/data/artefacts/us_local/run_sp500_skip.log

# Each takes ~2–5 minutes. Run both in parallel (separate tmux panes).

Key Arguments Reference

ArgumentDefaultDescription
--train_start2010-01-01Start of training data window
--as_oftodayScoring reference date (end of data)
--n_jobs1Parallel Optuna trials. Use 4 for NSE, 2 for SP500
--n_foldsautoCV folds — auto-computed: as_of_yr − start_yr − 2 + 1. 2010→2023 = 12 folds (test windows 2012–2023). Do not override.
--n_trials25Optuna HPO trials. 25 = good balance; 50 = exhaustive
--skip_trainoffSkip training, score only using saved models
--top_n30Stocks per watchlist

11. Monitor & Troubleshoot

Watch logs in real time

# Tail the log from another tmux pane
tail -f /mnt/data/artefacts/nse_local/run_nse_full.log
tail -f /mnt/data/artefacts/us_local/run_sp500_full.log

Check CPU & memory

htop   # H key to show/hide threads
What you seeMeaning
4–5 Python processes, ~30–35% CPU eachHPO running normally (4 parallel trials)
1 Python process, ~200–400% CPUFold caching running (sequential, LightGBM multithreaded)
All processes near 0% CPU for >10 minLikely hung — check log for errors

Common errors & fixes

ErrorCauseFix
OSError: No space left on device Disk full — checkpoints too large Delete *.pkl checkpoints and Optuna .db files; resize volume
Best iteration: 0 on every fold No val set passed to LightGBM early stopping Already fixed in current code — pull latest
ModuleNotFoundError: numpy._core Stale .pkl saved with different numpy version Delete all *.pkl checkpoints and rerun
Fold cache hangs forever (no output) Thread-unsafe recompute_zones Fold caching is sequential in current code — pull latest
Output folder empty after training Scoring used wrong panel (before feature engineering) Already fixed — pull latest
Git pull fails: divergent branches Local commits diverged from origin git stash → git reset --hard origin/master → git stash pop
