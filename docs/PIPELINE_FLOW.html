Architecture Overview

How the pipeline works

Eight sequential layers transform 499 NSE stock CSVs into two ranked watchlists — one for momentum setups, one for reversal setups — with separate ML models trained on each universe.

Layer 0

Data Ingestion

Raw files are loaded in parallel, merged into one master panel, and validated before any processing begins.

LAYER 0 Data Loading & Master Panel
Stock CSVs
  • 499 NSE tickers from constituentsi.csv
  • Daily OHLCV format, minimum 252 bars
  • Loaded in parallel via thread pool
  • ~457 tickers pass quality gate
Benchmark
  • ^NSEI daily close
  • Used for excess return targets
  • Used for NDCG validation metric
  • Used for portfolio Sharpe calculation
Master Panel
  • MultiIndex: (date, ticker)
  • ~1.2M rows · date range 2010→now
  • Columns: open, high, low, close, volume
  • Sector from CSV or defaults to NSE
⚠️ Tickers with no date column, fewer than 252 bars, or CSV parse errors are skipped with a WARNING and excluded from all downstream processing. MemoryError and PermissionError are raised immediately — they are not silently swallowed.

  • Per-TF flags: ssz_1wk/1mo/3mo/1y

  • ssz_htf_score — trend-confirmed composite

Layer 2

Target Building ✓ checkpoint

For each (date, ticker) row, the pipeline looks forward in time to compute return-based labels used as training targets. The composite rank blends three horizons to reward consistency.

LAYER 2 Target Building — pipeline/targets/builder.py 3 horizons
Forward Returns
  • future_20d_return (1 month)
  • future_40d_return (2 months)
  • future_60d_return (3 months)
  • future_20d_excess_return vs NSEI
  • max_drawdown_20d
  • future_vol_20d (annualised)
Cross-Section Ranks
  • cs_rank_20d — percentile [0,1] vs all stocks same date
  • cs_rank_40d — same for 40d horizon
  • cs_rank_60d — same for 60d horizon
  • 1.0 = best performer that day
  • 0.0 = worst performer that day
Composite & Labels
  • cs_rank_composite = 50% × 20d + 30% × 40d + 20% × 60d
  • Rewards stocks that outperform consistently
  • top_quintile = top 20% on date (bull label)
  • bot_quintile = bottom 20% on date (bear label)
  • Most recent ~20 bars have NaN (no future data yet)
# Composite rank — blends consistency across horizons
cs_rank_composite = 0.50 × cs_rank_20d + 0.30 × cs_rank_40d + 0.20 × cs_rank_60d

# Bull ranker target (momentum mode)
y_train = cs_rank_composite

# Bear ranker target (reversal mode) — inverted so declines rank highest
y_train = 1.0 − cs_rank_composite
Layer 3

Walk-Forward Cross Validation

The 16-year history is split into 12 expanding folds. Each fold trains on all history up to a cutoff and validates on the next year. Zones are recomputed fresh per fold to prevent future data leaking into zone boundaries.

LAYER 3 Purged Walk-Forward CV — pipeline/validation/cv.py 12 folds · no lookahead
Fold Structure
Fold 0   Train 2010→2012  |  Test 2012→2013  (skipped — insufficient history)
Fold 1   Train 2010→2013  |  Test 2013→2014
Fold 2   Train 2010→2014  |  Test 2014→2015
          ⋮
Fold 10   Train 2010→2022  |  Test 2022→2023
Fold 11   Train 2010→2023  |  Test 2023→2024
Guards
  • min_train_window = 504 bars (2 yrs)
  • Zones recomputed per fold cutoff
  • Fold cache: zones computed once, reused across 50 HPO trials
  • avg_group_size < 10 → fold skipped
Layer 4

Optuna Hyperparameter Optimisation

50 trials search the hyperparameter space using TPE sampler. Each trial runs across all valid folds. Pruning eliminates poor trials early. Progress is persisted to SQLite so the run can be resumed after a crash.

LAYER 4 Optuna HPO — 50 trials per mode slowest step
Search Space
  • num_leaves: 20–200
  • learning_rate: 0.005–0.2
  • n_estimators: 100–1000
  • min_child_samples: 10–100
  • subsample: 0.5–1.0
  • colsample_bytree: 0.4–1.0
  • reg_lambda: 0.001–10
  • feature_top_K: 20/30/40/50
Objective & Pruning
  • Maximize: mean NDCG@10 − 0.5 × std
  • Penalises unstable trials
  • Prune if top-decile excess ≤ 0
  • Prune if fewer than 3 valid folds
  • MedianPruner: 10 startup trials, 2 warmup steps
  • Saves to optuna_hpo.db (resumable)
Feature Pre-selection
  • FeatureSelector runs ONCE before HPO
  • Top-50 features selected via permutation importance + SHAP stability
  • Zone/ICT features force-included via ALWAYS_INCLUDE
  • HPO trials slice this pool (top_K = 20/30/40/50)
Layer 5

Two Independent Rankers

Two LGBMRanker models are trained on different universes with different targets. The momentum ranker learns what makes stocks go up. The reversal ranker learns what makes stocks go down.

📈 Momentum Ranker

Universe: stocks within 15% of 52w high (high_52w_dist > −0.15)

Target: cs_rank_composite — rank by highest future returns

Label: top_quintile for FeatureSelector

Learns: breakout continuation, demand zone entries near highs, ICT bull OB setups

Saves to: artefacts/nse_local/momentum/

📉 Reversal Ranker

Universe: stocks 15%+ below 52w high (high_52w_dist ≤ −0.15)

Target: 1 − cs_rank_composite — rank by lowest future returns (declines first)

Label: bot_quintile for FeatureSelector

Learns: supply zone rejections, SSZ entries, ICT bear OB setups, breakdown patterns

Saves to: artefacts/nse_local/reversal/

LAYER 5 FeatureSelector — pipeline/selection/selector.py
Step 1 — NaN Filter
  • Drop features with >5% missing values
  • Zone/ICT features kept even if NaN-heavy (imputed to 0)
Step 2 — Correlation Pruning
  • Drop lower-importance feature in pairs with |ρ| > 0.92
  • Zone features get 2.5× importance boost to win ties
Step 3 — Permutation Importance
  • Rank remaining features on 20% held-out slice
  • Take top-K ordinary features
Step 4 — SHAP Stability
  • 3 bootstrap resamplings
  • Flag features with rank std > 10 as unstable
  • Zone/ICT features never flagged
🔒 ALWAYS_INCLUDE — these features bypass all elimination: sdz_htf_score, ssz_htf_score, sdz_1wk/1mo/3mo/1y, ssz_1wk/1mo/3mo/1y, dz_raw_score, sz_raw_score, zone_htf_confluence, all ICT OB/FVG/BB active + dist features, BSL/SSL sweeps.
Layer 6

Scoring & Watchlist Construction

All 457 stocks are scored on the latest date. The final score blends ML model output with a zone-only composite. Only stocks with an active SDZ or SSZ can appear on the watchlist.

LAYER 6 score_and_rank() — run_nse_local.py
# Momentum mode
bull_score = norm( 0.70 × model_score + 0.30 × SDZ_composite )
bear_score = norm( 0.70 × (1 − model_score) + 0.30 × SSZ_composite )

# Reversal mode
bull_score = norm( 0.65 × model_score + 0.35 × SDZ_composite )
bear_score = norm( 0.65 × (1 − model_score) + 0.35 × SSZ_composite )

sdz_1mo          w = 2.5
sdz_3mo          w = 3.0
sdz_1y           w = 3.5
Bear Composite (SSZ only)
ssz_htf_score    w = 3.0
ssz_1wk          w = 2.0
ssz_1mo          w = 2.5
ssz_3mo          w = 3.0

🔍 Zone presence filter: stocks with composite score = 0 (no active SDZ/SSZ signal) are excluded from the watchlist even if their model score is high. This enforces the design principle: every pick must have a zone backing it.

Runtime

Run Times & Commands

Feature and target checkpoints mean most steps only run once. Subsequent runs reuse them and go straight to training.

L1
Feature Engineering
Runs only when panel_features.pkl is absent or deleted. Processes 457 tickers with ICT, zone, and multi-TF signals.
~3 min · once per data refresh
L2
Target Building
Computes forward returns and cross-section ranks for all 1.2M rows. Cached to panel_targets.pkl.
~1 min · once
L3
Fold Zone Pre-computation
Zones recomputed per fold cutoff for each mode. Cached in memory for the HPO run. Cannot be skipped — zones must be honest about what was visible at each training cutoff.
~20 min · per mode
L4
HPO — 50 Optuna trials
Each trial trains across all valid folds. MedianPruner eliminates weak trials early. Progress saved to SQLite — resume anytime without losing completed trials.
~40 min · per mode
L5
Final Model Training
FeatureSelector + LGBMRanker on full filtered dataset. EnsembleRanker validation on held-out folds.
~3 min · per mode
L6
Scoring + Output
Score all 457 stocks, apply zone filter, construct 4 watchlists, generate SHAP + HTML report.
~1 min · per mode
Full Run (first time)
python run_nse_local.py --mode all
# ~90 min total
Re-score only (no retrain)
python run_nse_local.py --skip_train
# ~2 min
Quick smoke test
python run_nse_local.py --mode all \
  --n_trials 0 --n_folds 3
# ~15 min
Custom watchlist size
python run_nse_local.py --skip_train \
  --top_n 15
